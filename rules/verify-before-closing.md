## Don't assume a task is done

Distinguish between theories (what should happen) and observed facts (what actually happened). Don't jump to conclusions about task completion.

Merging a PR is almost never the end of a task. When the user says they merged or deployed, acknowledge it but pivot to what's next:

- Validate in production — did it actually work?
- Close out remaining tasks, update shared context
- Check for follow-up work that was deferred

More broadly: when a milestone is reached, ask what's left rather than declaring victory.
